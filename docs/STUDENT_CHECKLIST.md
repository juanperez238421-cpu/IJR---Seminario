# Lista de verificación para estudiantes

Usa esta lista para revisar tu proyecto antes de presentarlo.

## HTML

- [ ] El archivo principal se llama `index.html`.
- [ ] La página usa etiquetas semánticas como `header`, `main`, `section`, `aside` y `footer`.
- [ ] Todos los campos tienen un `label` asociado.
- [ ] Los botones tienen un propósito claro y usan el tipo correcto.
- [ ] Los archivos CSS y JavaScript están enlazados mediante rutas relativas.

## CSS

- [ ] La interfaz puede usarse en computador y celular.
- [ ] No hay textos, botones ni tarjetas superpuestos.
- [ ] Los estados `hover`, `focus` y `disabled` son visibles.
- [ ] Los nombres de clases describen su función.
- [ ] Los estilos están en un archivo CSS externo.

## JavaScript

- [ ] El catálogo se genera desde datos y no repitiendo manualmente todas las tarjetas en HTML.
- [ ] Los eventos se registran desde JavaScript.
- [ ] El carrito conserva producto, precio y cantidad.
- [ ] Cambiar una cantidad actualiza los totales.
- [ ] El usuario puede eliminar productos y vaciar el carrito.
- [ ] El formulario valida los datos antes de continuar.
- [ ] Los cálculos importantes se realizan mediante funciones reutilizables.
- [ ] La aplicación muestra mensajes comprensibles cuando ocurre un error.

## Persistencia y salida

- [ ] El estado se conserva al recargar la página.
- [ ] La orden genera un comprobante legible.
- [ ] El comprobante puede imprimirse.
- [ ] Reiniciar la aplicación elimina únicamente los datos propios del proyecto.

## Calidad del repositorio

- [ ] El repositorio tiene un `README.md` claro.
- [ ] No se subieron contraseñas, tokens ni información privada.
- [ ] Los nombres de archivos y carpetas son consistentes.
- [ ] La rama `main` contiene una versión estable.
- [ ] La página de GitHub Pages funciona sin errores en la consola.
- [ ] Las pruebas terminan correctamente con `npm test`.

## Explicación oral

Debes poder explicar, sin leer el código completo:

1. Qué estructura define el HTML.
2. Cómo el CSS organiza y adapta la interfaz.
3. De dónde salen los productos.
4. Cómo se actualiza el carrito.
5. Cómo se calculan los totales.
6. Cómo se valida el formulario.
7. Qué guarda `localStorage`.
8. Qué mejorarías en una siguiente versión.
